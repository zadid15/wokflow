Buat table order_items :
- order_item_id

- menu_id (foreign key ke table menus.menu_id)
- qty (integer)
- status enum (waiting_queue, cooking, done, served, cancelled)
- created_at
- updated_at

Buat table attributes :
- attribute_id
- name
- created_at
- updated_at

Buat table order_item_attributes :
- order_item_attribute_id
- order_item_id (foreign key ke table order_items.order_item_id)
- attribute_id (foreign key ke table attributes.attribute_id)
- value (boolean)
- created_at
- updated_at

Business Logic :
    1. source wajib dan harus salah satu dari : cashier, whatsapp.

    2. Masing masing source auto generate queue_number, dengan format penomoran antrean sebagai berikut :
        - Cashier : menggunakan format C-nomorurut,
        - WhatsApp : menggunakan format W-nomorurut

        Contoh :
        - Jika order pertama maka nomor urutnya adalah 1, jadi queue_number nya adalah C-1 atau W-1
        - Jika terapat 2 orderan dimana 1 orderan dari WhatsApp (orderan pertama) dan 1 orderan dari Cashier (orderan kedua) maka :
            - Orderan pertama format antreannya menjadi W-1
            - Orderan kedua format antreannya menjadi C-1
        - Nomor urut queue_number dihitung terpisah berdasarkan source :
            - Cashier memiliki counter sendiri
            - WhatsApp memiliki counter sendiri
        - queue_number harus di-generate secara atomic menggunakan database transaction atau locking mechanism untuk menghindari duplikasi nomor.

    3. Jika source = cashier :
        - customer_name boleh nullable
        - customer_phone boleh nullable

    4. Jika source = whatsapp :
        - minimal salah satu dari customer_name atau customer_phone wajib diisi

    5. Urutan order ditentukan berdasarkan created_at

    6. queue_number harus di generate secara atomic

    7. Semua proses create dilakukan dalam 1 database transaction

    8. Penghapusan menggunakan soft delete (deleted_at)

    9. Gunakan artisan
